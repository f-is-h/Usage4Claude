# Usage4Claude 产品网站开发指南

> 本指南用于指导使用Claude Code开发Usage4Claude的产品网站  
> 适用于Cloudflare Pages部署的静态网站项目

---

## 📋 项目概述

### 项目目标
为Usage4Claude macOS应用创建一个现代、轻量、专业的产品展示网站，主要用途：
1. 向潜在用户介绍产品功能和特性
2. 提供下载和安装指南
3. 满足Stripe支付审核要求（包含特定商取引法表記等法律页面）
4. 作为项目的官方门户网站

### 核心要求
- ✅ **现代轻量**：简洁优雅的设计，快速加载
- ✅ **完全免费**：不使用任何付费组件或服务
- ✅ **LLM友好**：易于AI理解和维护的代码结构
- ✅ **合规性**：满足日本特定商取引法和Stripe审核要求
- ✅ **易部署**：适配Cloudflare Pages的静态网站架构

---

## 🛠 技术栈选择

### 推荐技术栈
```
HTML5                    - 页面结构
Tailwind CSS (CDN)       - 样式框架（苹果风格UI）
Alpine.js (可选)         - 轻量级交互（仅在需要时使用）
纯CSS动画                - 视觉效果
无构建工具               - 直接部署
```

### 为什么选择这个技术栈？
1. **零构建步骤**：静态文件直接部署到Cloudflare Pages
2. **LLM最擅长**：纯HTML/CSS/JS是AI最熟悉的技术
3. **性能最优**：无框架开销，加载速度极快
4. **维护简单**：没有依赖版本问题，易于长期维护
5. **完全免费**：所有工具和CDN都是开源免费的

## 🎨 设计原则与风格指南

### 整体设计理念
**参考苹果产品页面的设计风格**：简洁、优雅、专注内容

### 视觉设计要点

#### 1. 配色方案
- **主色调**：基于Usage4Claude的品牌色
  - 蓝色系（5小时限制）：#007AFF
  - 紫色系（7天限制）：#5E5CE6
  - 背景色：白色/浅灰 (#F5F5F7)
- **状态色**：绿色（安全）、橙色（警告）、红色（限制）
- **文字**：深灰色为主，避免纯黑

#### 2. 排版设计
- **字体栈**：使用系统原生字体
  ```css
  font-family: -apple-system, BlinkMacSystemFont, 'SF Pro Display',
               'Segoe UI', 'Hiragino Sans', sans-serif;
  ```
- **字号层级**：
  - 大标题：48px-72px（Hero区）
  - 中标题：32px-48px（功能模块）
  - 正文：16px-18px
  - 辅助文字：14px

#### 3. 间距与留白
- 使用Tailwind的spacing scale（4px基准）
- 保持充足的留白，避免拥挤感
- 模块间距：80px-120px

#### 4. 响应式设计
- 移动优先（Mobile First）
- 断点：sm(640px), md(768px), lg(1024px), xl(1280px)
- 确保所有内容在移动端可读可用

## 📄 页面结构与内容规划

### 首页（index.html）结构

#### 必需的内容模块（按顺序）：

1. **导航栏（Header）**
   - Logo/应用名称
   - 主要导航链接（功能、下载、GitHub、Support）
   - 语言切换（如支持多语言）

2. **Hero区**
   - 主标题：产品名称和slogan
   - 副标题：一句话介绍核心价值
   - 主要CTA按钮：下载、GitHub Star
   - Hero图：应用截图或演示视频

3. **核心功能展示**
   - 3-6个主要功能点
   - 每个功能：图标 + 标题 + 简短描述
   - 基于README中的Features部分提炼

4. **产品截图展示**
   - 菜单栏显示样式
   - 详情窗口截图
   - 设置界面展示
   - 使用场景演示

5. **技术亮点**
   - 跨平台支持（Web、Claude Code、Desktop、Mobile）
   - 多语言支持
   - 安全与隐私保护
   - 开源透明

6. **下载/安装区**
   - 系统要求说明
   - 下载按钮（链接到GitHub Releases）
   - 安装步骤简述
   - GitHub Star按钮

7. **页脚（Footer）**
   - 版权信息：MIT License
   - 链接：GitHub、Issues、Discussions
   - 法律链接：特商法表記、Privacy Policy
   - 联系方式：Email（可选）

### 特商法页面（legal.html）结构

#### 必需内容（日英双语）：

**页面配置**：
```html
<meta name="robots" content="noindex, nofollow">
```
（防止搜索引擎索引，保护隐私）

**必需信息项**：
1. 页面标题：特定商取引法に基づく表記 / Notation based on the Act on Specified Commercial Transactions
2. 事業者名（Business Name）：开发者真实姓名
3. 所在地（Address）：[部署时填写完整地址，源码中使用占位符]
4. 連絡先（Contact）：Email地址
5. 商品・サービスの内容（Product/Service）：说明这是开源软件的自愿捐赠
6. 価格（Price）：任意捐赠金额，软件本身免费
7. 支払方法（Payment Method）：GitHub Sponsors (via Stripe)
8. 返品・キャンセル（Returns/Cancellation）：说明捐赠性质，一般不退款
9. ライセンス（License）：MIT License

**设计要点**：
- 使用小字号（text-sm）和低对比度颜色
- 保持信息完整但不过于显眼
- 提供日语和英语两个版本

### 隐私政策页面（privacy.html）

**核心说明事项**：
1. 数据收集：明确说明"不收集任何用户数据"
2. 本地存储：所有数据仅存储在用户本地Mac
3. Keychain使用：解释Session Key的加密存储
4. 无追踪：不使用任何分析或追踪工具
5. 开源透明：代码公开，可供审计
6. 联系方式：提供反馈渠道

**必须分阶段开发，每个阶段完成后手动验证再继续！**

不要一次性开发所有页面，应该：
1. 完成一个阶段
2. 在浏览器中打开HTML文件手动测试
3. 确认效果满意后再进行下一阶段
4. 发现问题立即修复，不要累积问题

## 📦 Cloudflare Pages部署配置

### 部署前准备

**在website目录下创建README.md**，包含以下信息：

1. **项目说明**
2. **本地预览方法**
3. **部署步骤**
4. **地址占位符说明**

### Cloudflare Pages配置

**构建配置**：
```yaml
Build command: (留空)
Build output directory: /
Root directory: website
```

**环境变量**：无需配置

### 部署流程

1. **连接GitHub仓库**
   - 登录Cloudflare Dashboard
   - 创建新的Pages项目
   - 连接到Usage4Claude仓库
   - 选择website目录作为根目录

2. **替换地址占位符**
   - 在Cloudflare Pages的文件编辑中
   - 将legal.html中的`[ADDRESS_PLACEHOLDER]`
   - 替换为真实地址

3. **自定义域名**（可选）
   - 在Cloudflare Pages设置中
   - 添加自定义域名
   - 配置DNS记录

4. **测试部署**
   - 访问生成的.pages.dev URL
   - 验证所有页面正常工作
   - 检查legal页面的地址信息

## 💡 最佳实践提醒

### 对Claude Code的建议

1. **分阶段执行**
   - 不要试图一次完成所有内容
   - 每完成一个阶段就停下来让用户验证
   - 根据反馈调整后续计划

2. **保持代码简洁**
   - 避免过度工程
   - 不需要复杂的JavaScript逻辑
   - 优先使用Tailwind的utility classes

3. **注重细节**
   - 间距要一致
   - 对齐要精确
   - 颜色要和谐
   - 文字要清晰

4. **测试为先**
   - 每个改动都在浏览器中验证
   - 在不同设备和浏览器上测试
   - 检查响应式布局

5. **文档清晰**
   - 代码注释清楚
   - HTML结构语义化
   - CSS类名有意义
